# Evidence Quality Bar

Use this rubric to label artifact strength.

## Strong

The artifact:
- Comes from the source system
- Is dated or clearly scoped to the audit period
- Shows the control operating
- Can be reproduced later

Examples:
- IdP export from the current quarter
- Ticket showing access review completed with approver
- Restore test record with timestamp and result

## Acceptable

The artifact:
- Is probably sufficient for readiness review
- Shows the control or activity with moderate confidence
- May need one supporting artifact later

Examples:
- Screenshot plus linked ticket
- Dashboard export without full audit trail

## Weak

The artifact:
- Shows intent more than execution
- Lacks date, owner, or period context
- Cannot be easily re-traced

Examples:
- Undated screenshot
- Notes that say "done" with no source link

## Missing

There is no artifact or the artifact does not support the stated control.

When in doubt, rate down rather than up.
